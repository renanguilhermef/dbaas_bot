How to backup a database with RMAN when an archivelog is corrupted

Summary


RMAN backup including the archivelogs fails with:

Corrupt block seq: 32326 blocknum=1.
Bad header found during deleting archived log
Data in bad block - seq:0. bno:0. time:0
beg:0 cks:0
calculated check value: 0

Solution


 Check the archivelog if it is corrupted:

       RMAN> validate archivelog sequence 32326 thread 2;

1) Delete the archivelog manually

2) Execute the following rman command.

       RMAN > crosscheck archivelog all;

3) Now run the RMAN delete command .

      RMAN> DELETE EXPIRED ARCHIVELOG sequence 32326;

 

4) Take an RMAN full backup

 


Attachments :

Cause


 


 Archivelog sequence 32326 is corrupted.
